BUCLES

En JavaScript, los bucles o iteraciones son estructuras de control que permiten ejecutar un bloque de código de manera repetida, dependiendo de una condición.

Tipos de Bucles

  • For

    • Es el bucle más común y se utiliza cuando sabes de antemano cuántas veces necesitas iterar.

                        
                        for (var i = 0; i < 9; i++){
                            console.log(i)
                        }

    Explicación

    • var i = 0:
    • i < 9: La condición que se evalúa antes de cada iteración
    • i++: Modifica la variable en cada paso
  • While

    • Este bucle ejecuta el código mientras una condición sea verdadera. El número de iteraciones no necesita estar predeterminado.

                    
                    let i = 1;
                    while (i<= 5){
                        console.log(i)
                        i++
                    }

    Explicación

    • let i
    • while (i<= 5): Ejecuta mientras i sea menor que 5
    • i++: Incrementa i en 1
  • do...while

    • Similar al bucle while, pero en este caso la condición se evalúa después de ejecutar el código. Esto asegura que el código se ejecute al menos una vez, incluso si la condición es falsa desde el principio. 

                    
                    let i = 1;
                    do{
                        console.log(i);
                        i++;
                    } 
                    while (i<=5);

    Explicación

    • do: Realiza el bucle al menos una vez
    • while (i<=5): Hasta que se cumpla una vez.
  • for...in

    • Este bucle se utiliza para iterar sobre las propiedades de un objeto. Es útil cuando necesitas recorrer las claves de un objeto.

Maps

  • Un mapa contiene pares de valores llave "Key-Value" las cuales pueden ser de cualquier tipo.

  • Un mapa recuerda el orden original de inserción de las llaves.

  • Se puede añadir "set" y quitar "delete" elementos de un mapa.

Diferencias entre Maps y Objetos JS

    Maps
  • Iterable directamente
  • Tiene una propiedad de tamaño
  • Las key puede ser de cualquiertipo de dato
  • Las key están ordenadas por orden de inserción
  • No tiene keys predeterminadas
    Objetos
  • No se puede iterar directamente
  • No tiene una propiedad de tamaño
  • Las keys deben ser strings o simbolos
  • Las keys no están ordenadas
  • Tiene keys predeterminadas
Metodo Descripción
new Map() Crea un nuevo mapa
set() Dispone un value para una key
size Devuelve el numero de elementos en un Map
clear() Borra todos los elementos de un mapa
delete() Borra un elemento específico
entries() Devuelve un iterador con los pares [key, value]
get() Toma un value para una key
groupBy() Agrupa elementos de objetos acorde con los valores de respuesta devueltos
has() Devuelve true si la key existe en el Map
keys() Devuelve un iterador con la key del Map
values() Devuelve un iterador con el value
foreach()

Convoca una respuesta por cada par key/value

Convoca una función por cada elemento

No cambia el Map original
